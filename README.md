## Цель работы

Освоить базовые принципы виртуализации ОС на примере установки Debian и развернуть виртуальный веб-сервер по стеку LAMP с двумя сайтами:

* `phpmyadmin.localhost`
* `wordpress.localhost`

## Ход выполнения работы

### Этап 1. Подготовка среды в PowerShell

1. Загружен серверный ISO-образ Debian (x64, без графической оболочки).
2. Установлен гипервизор **QEMU**.
3. Создан Git-репозиторий `cv-lab2-lamp` и выполнено клонирование на локальную машину:

```bash
git clone https://github.com/USM-Labs/cv-lab2-lamp
```

4. Под ISO-образ создан каталог `dvd`.
5. ISO Debian помещен в `dvd` и переименован в `debian.iso`.
6. Добавлен `.gitignore` с файлами, которые не должны попадать в репозиторий:

```gitignore
*.qcow2
*.iso
*.zip
```

7. Сформирован виртуальный диск `qcow2` на **8 ГБ**:

```bash
qemu-img create -f qcow2 debian.qcow2 8G
```

8. Запущен процесс установки ОС:

```bash
qemu-system-x86_64 -hda debian.qcow2 -cdrom dvd/debian.iso -boot d -m 2G
```

9. В ходе установки указаны параметры:

* **Имя компьютера:** `debian`
* **Домен:** `debian.localhost`
* **Пользователь:** `user`
* **Пароль пользователя:** `password`
* **Пароль root:** не установлен

10. После завершения установки виртуальная машина запущена следующей командой:

```bash
qemu-system-x86_64 -hda debian.qcow2 -m 2G -smp 2 -device e1000,netdev=net0 -netdev user,id=net0,hostfwd=tcp::1080-:80,hostfwd=tcp::1022-:22
```

### Этап 2. Настройка LAMP-сервера

1. Выполнен вход в систему под пользователем `user`.
2. Получены права администратора:

```bash
sudo -i
```

3. Обновлены индексы пакетов:

```bash
apt update -y
```

4. Установлены компоненты стека LAMP:

* Apache
* PHP
* MariaDB
* необходимые модули

5. Скачаны дистрибутивы:

* phpMyAdmin
* WordPress

6. Создан каталог `/var/www` и распакованы архивы:

* phpMyAdmin - в `/var/www/phpmyadmin`
* WordPress - в `/var/www/wordpress`

### Этап 3. Создание базы данных

1. Открыта консоль MariaDB:

```bash
mysql -u root
```

2. Созданы база и пользователь, выданы права на БД:

```sql
CREATE DATABASE wordpress_db;
CREATE USER 'user'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON wordpress_db.* TO 'user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### Этап 4. Настройка виртуальных хостов Apache

1. В каталоге `/etc/apache2/sites-available/` подготовлены два конфигурационных файла:

* `01-phpmyadmin.conf`
* `02-wordpress.conf`

2. В каждом конфиге задано:

* `DocumentRoot`
* `ServerName`
* `ServerAlias`
* пути к логам

3. Конфигурации подключены командами:

```bash
a2ensite 01-phpmyadmin

a2ensite 02-wordpress
```

4. В файл `/etc/hosts` добавлены записи для локальных доменов:

```text
127.0.0.1 phpmyadmin.localhost
127.0.0.1 wordpress.localhost
```

5. Apache перезапущен:

```bash
systemctl restart apache2
```

6. Проверено состояние сервиса:

```bash
systemctl status apache2
```

### Этап 5. Проверка работы сайтов

1. В браузере открыты страницы:

* `http://wordpress.localhost:1080`
* `http://phpmyadmin.localhost:1080`

2. Для установки WordPress указаны параметры БД:

* **База данных:** `wordpress_db`
* **Пользователь:** `user`
* **Пароль:** `password`

3. В phpMyAdmin выполнен вход с теми же учетными данными.

Результат: оба сайта корректно запускаются и доступны по заданным адресам.

## Ответы на вопросы

### 1. Что такое виртуализация?

Виртуализация - это подход, при котором создается виртуальная среда для вычислительных ресурсов (ОС, серверов, дисков, сети). Благодаря этому на одном физическом компьютере можно запускать несколько операционных систем.

### 2. Что такое гипервизор?

Гипервизор - это программный слой, который создает и управляет виртуальными машинами, а также распределяет ресурсы между ними. В рамках данной работы использовался QEMU.

### 3. Что такое LAMP?

LAMP - это типовой набор серверного ПО:

* **L** - Linux (в данной работе Debian)
* **A** - Apache (веб-сервер)
* **M** - MariaDB/MySQL (СУБД)
* **P** - PHP (язык серверной разработки)

Стек применяется для разработки и размещения веб-приложений.

### 4. Что такое Apache?

Apache - это веб-сервер, принимающий HTTP-запросы и возвращающий клиенту веб-страницы и другой контент.

### 5. Для чего нужен phpMyAdmin?

phpMyAdmin - это веб-интерфейс для администрирования MySQL/MariaDB: создание и настройка баз и таблиц, а также выполнение SQL-запросов через браузер.

### 6. Для чего нужен WordPress?

WordPress - CMS для создания сайтов и блогов. Она позволяет развернуть сайт без ручной разработки всего функционала с нуля.

### 7. Что делает файл /etc/hosts?

Файл `/etc/hosts` сопоставляет доменные имена с IP-адресами. В этой работе он использовался, чтобы обращаться к сайтам по именам `phpmyadmin.localhost` и `wordpress.localhost`.
