## Краткая документация

- `Docker Desktop` - приложение для запуска Docker Engine на локальной машине.
- `Dockerfile` - файл с инструкциями для сборки образа.
- `site/index.html` - пример статической страницы, которая могла бы копироваться в контейнер.
- `docker build` - команда для сборки образа.
- `docker run` - команда для запуска контейнера из образа.

Предполагаемая структура проекта при практическом выполнении:

- `Dockerfile` - описание образа на базе `debian:latest`.
- `site` - каталог со статическими файлами сайта.
- `README.md` - отчет по выполнению лабораторной работы.

## Примеры использования

Пример команды сборки:

```text
docker build -t containers02 .
```

Пример содержимого `Dockerfile`:

```dockerfile
FROM debian:latest
COPY ./site/ /var/www/html/
CMD ["sh", "-c", "echo hello from $HOSTNAME"]
```

Пример входных данных:

```text
site/index.html
Dockerfile
```

Пример результата:

```text
hello from 8f4d9c2a1b7e
```

## Описание выполнения

1. Docker Desktop скачивается с официального сайта Docker и устанавливается стандартным способом.
2. После запуска Docker Desktop проверяется доступность Docker Engine командой `docker --version`.
3. Создается учебный проект `containers02`.
4. В проект добавляется `Dockerfile`, который использует образ `debian:latest`, копирует файлы сайта в `/var/www/html/` и выводит имя контейнера.
5. Командой `docker build -t containers02 .` выполняется сборка образа.
6. Командой `docker run --name containers02 containers02` запускается контейнер.
7. Для интерактивной проверки контейнер удаляется и запускается повторно с командой `bash`.

## Ответы на вопросы

1. Сколько времени создавался образ?

   Для типовой локальной машины сборка такого образа занимает около 10-25 секунд при первом запуске, так как Docker загружает базовый образ `debian:latest`. При повторной сборке с использованием кэша время может уменьшиться до 1-3 секунд.

2. Что было выведено в консоли при запуске контейнера?

   Контейнер выводит строку вида:

   ```text
   hello from 8f4d9c2a1b7e
   ```

   Последняя часть строки - имя хоста контейнера, которое Docker генерирует автоматически.

3. Что выводится после перехода в `/var/www/html/` и выполнения `ls -l`?

   Ожидаемый вывод содержит файл `index.html`, скопированный из каталога `site`:

   ```text
   total 4
   -rw-r--r-- 1 root root 186 Apr 18 12:14 index.html
   ```

## Безопасность

В данной работе контейнер не принимает пользовательский ввод и не публикует сетевой сервис. Основной аспект безопасности - понимание того, что контейнер запускается из образа, а файлы внутрь него попадают только через явно указанные инструкции `COPY`.
